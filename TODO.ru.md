## IPv6 диагностика и настройка

Проверка IPv6 показала, что стек работает, но глобальные адреса отсутствуют и внешняя связь невозможна.

**Проблемы:**
- ❌ Глобальные IPv6‑адреса отсутствуют (только link‑local `fe80::`)
- ❌ Маршрут по умолчанию (`default`) не настроен
- ❌ Связь с внешними IPv6‑хостами невозможна (`Network is unreachable`)
- ✅ DNS разрешение AAAA‑записей работает
- ❌ Туннельные механизмы (6to4, Teredo, Miredo) не установлены
- ⚠️  `ip6tables` активен с политикой `DROP` на INPUT/FORWARD (может блокировать ICMPv6)

**Действия:**
1. Проверить настройки IPv6 на маршрутизаторе/коммутаторе (включить SLAAC/DHCPv6)
2. Если провайдер не предоставляет IPv6, настроить туннель (Hurricane Electric, SixXS)
3. При необходимости временно отключить `ufw` для IPv6 (`sudo ufw disable` для IPv6) или добавить правила для ICMPv6
4. Для тестирования можно использовать `ping -6` с указанием интерфейса (`-I`) или `curl --interface`

## wl-daemon: переподключение при broken-pipe вместо graceful shutdown

> **Работа в вышестоящем репозитории** (`github.com/neg-serg/wl`, Rust код) — не задача Salt-репозитория.
> Оставлено здесь как заметка для отслеживания.

Когда Wayland compositor выполняет быстрое удаление и повторное добавление выхода (hotplug монитора, смена режима, `hyprctl reload`, цикл DPMS), `wl-daemon` может успеть выполнить `wl_display_flush()` на фоне уже закрытого `wl_output`. Flush возвращает `EPIPE` (Broken pipe), ошибка распространяется как `wayland flush error` в главный цикл, и демон в настоящее время корректно завершает работу (exit status 0).

**Правильное исправление (upstream):**
- [x] Обрабатывать ошибки dispatch/flush как восстанавливаемые — вызывать `reconnect_wayland()` вместо `break`
- [ ] Добавить ограниченный повтор (5 попыток за 10с) внутри цикла переподключения
- [ ] Выводить понятную строку лога: `wayland connection lost, reconnecting (attempt N/5)`
- [x] Оставить `Restart=always` в wl.service как защиту в глубину

**Вторичная очистка** (косметическая): retry-цикл `ExecStartPost=wl restore` в unit-файле срабатывает до готовности IPC-сокета демона. Исправление: `sd_notify(READY=1)` + `Type=notify` + убрать sleep-based retry цикл.
