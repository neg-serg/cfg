# Настройка AI в Neovim

- Последнее обновление: 2026-04-23
- Статус: Активно (интеграция Gemini AI удалена)

## Текущие AI‑интеграции

### Автодополнение
- **blink.cmp**: LSP‑автодополнение с поддержкой:
  - Language Server Protocol (LSP)
  - Слова из буфера
  - Пути
  - Сниппеты (через LuaSnip)
- **Внешние AI‑провайдеры отсутствуют** (Google Gemini AI удалён)

### Удалённые компоненты
- **minuet-ai.nvim**: Плагин, предоставлявший автодополнение от Gemini AI — удалён
- **Конфигурация Gemini API** — удалена из `minuet.lua` и `blink.lua`
- **Правила ProxyPilot для Gemini** — удалены из `config.yaml`
- **Модели Gemini в OpenCode** — удалены из `opencode.json.tmpl`

## Файлы конфигурации

- `dotfiles/dot_config/nvim/lua/plugins/completion/blink.lua` — основная конфигурация автодополнения
- `dotfiles/dot_config/nvim/lua/plugins/completion/lspconfig.lua` — настройка LSP

## Добавление новых AI‑провайдеров

Чтобы добавить новый AI‑провайдер автодополнения:

1. Установите плагин (например, `llm.nvim`, `codeium.nvim` и т.д.)
2. Настройте его в `blink.lua` в разделе `sources`
3. Убедитесь, что необходимые API‑ключи хранятся в `gopass` и загружаются через переменные окружения
4. Обновите этот документ

## Примечания

- Группа состояний `ai` (`just group ai`) включает `opencode`, но больше не содержит состояний, связанных с Gemini.
- Управление секретами для AI (API‑ключи) продолжает использовать `gopass` с бэкендом `age`/`Yubikey`.
- Будущие AI‑интеграции должны следовать той же схеме: плагин → конфигурация → управление секретами.
